## Part 1: Diagnosis

Audited files: `frontend/app/search/page.tsx`, `backend/routers/movies.py`, `frontend/lib/api.ts`, `frontend/lib/tmdb.ts`, `frontend/app/globals.css`, `frontend/app/page.tsx`.

### A. UX limitations (why the page feels clunky)

1. **Discovery is four hardcoded lists and nothing else.** `search/page.tsx:30-35` defines exactly Trending / Popular {year} / Now Playing / Top Rated. There is no way to browse by genre, decade, rating floor, runtime, or streaming service — for a "what should we watch tonight" app, the single most common questions ("a good 90s comedy", "something short on Netflix", "highly-rated sci-fi") are unanswerable.
2. **"Popular 2026" is a misleading hardcode.** The label is built from `new Date().getFullYear()` (`search/page.tsx:28,32`) and the backend pins `primary_release_year` to the current year (`movies.py:31`). Five months into the year this is a thin, churn-y list, and all-time popular is simply unavailable.
3. **The genre chips lie.** Chips are derived only from the currently loaded results (`search/page.tsx:200`) and filtering is client-side over those loaded pages (`:201-203`). Consequences: chips appear/disappear as you press Load More; the count line (`:290,295`) counts loaded items, not actual matches; with "Comedy" selected, **Load More fetches 20 unfiltered movies of which maybe 2 are comedies** — the filtered grid grows by dribbles while the network does full-page work. The Load More button's visibility (`:372`, `page < totalPages`) is also computed from the *unfiltered* list, so it shows even when a filter has zero remaining matches.
4. **Category tabs vanish while a query is typed** (`search/page.tsx:250`, `{!isSearch && …}`), and clearing the query snap-resets to Trending page 1 via the browse effect (`:90-105`) — losing whatever browse state you had. Search and browse behave like two different pages awkwardly sharing one input.
5. **No URL state.** Query, category, genre, and pagination live only in `useState` (`:45-58`). Refresh, back button, or sharing a link all dump you back to default Trending. A filterable discovery page without URL state can't be bookmarked ("good horror on Netflix" is a view you'd want to revisit).
6. **No person search.** Typing "Tom Hanks" hits TMDB `/search/movie` (`movies.py:70-78`), which matches *titles* only — you get junk instead of his filmography. TMDB's `/search/person` + discover `with_people` is the standard solution and isn't proxied at all.
7. **Search ignores year.** TMDB `/search/movie` accepts `year`/`primary_release_year`; the proxy doesn't expose it (`movies.py:71`), so disambiguating "Dune 2021" vs "Dune 1984" relies entirely on TMDB's popularity ordering.

### B. Code-level flaws

8. **State sprawl: 15 `useState`/`useRef` hooks** (`search/page.tsx:45-62`) with four intertwined loading/mode booleans (`searching`, `browsing`, `loadingMore`, `isSearch`). Reset logic is duplicated across two effects (`:90-105` and `:108-133`), and `setSelectedGenre(null)` fires as a side effect at `:115` on *every keystroke* (before the debounce), so a genre filter dies the moment you touch the search box.
9. **Race conditions on browse and Load More.** The search path has a staleness guard (`currentQueryRef`, `:62,120-123`) but the browse effect (`:98-104`) has none — switch tabs quickly and a slow earlier response can overwrite the newer one. `loadMore()` (`:135-153`) reads `query`/`category`/`page` at call time with no guard and unconditionally appends (`:145`); fire it just before switching tabs and old-category pages get appended onto the new category's results.
10. **Duplicate React keys are possible.** TMDB popularity-sorted pages reshuffle between requests, so page N+1 can contain a movie already on page N. `setResults(prev => [...prev, ...data.results])` (`:145`) never dedupes → duplicate `key={movie.id}` warnings and double cards.
11. **~55 lines of card JSX duplicated from the For You page** (`search/page.tsx:310-362` vs `app/page.tsx:232-323`) — and they've already drifted: the For You card has the poster vote badge (`page.tsx:270-284`) while the search card renders the rating as plain text (`search/page.tsx:339`) and calls `movie.vote_average.toFixed(1)` unguarded, printing "⭐ 0.0" for unreleased films and `undefined ·` when `release_date` is empty.
12. **The debounce timeout is never cleaned up on unmount** (`:108-133` returns nothing), and `setResults([])` inside the timeout (`:118`) blanks the grid before the response arrives instead of dimming it.

### C. Wasted TMDB capability (backend)

13. **`movies.py` already has a `_discover` helper (`:18-25`) and uses it for exactly one hardcoded query** (`:28-34`). TMDB `/discover/movie` supports `with_genres`, `primary_release_date.gte/lte`, `vote_average.gte` + `vote_count.gte`, `with_runtime.gte/lte`, `with_watch_providers` + `watch_region`, `with_people`, `with_original_language`, and `sort_by` — one flexible endpoint can replace three of the four category endpoints *and* power a real filter UI.
14. **`/popular`, `/now_playing`, `/top_rated` (`movies.py:28-34, 48-67`) are all special cases of discover** (popularity sort, theatrical-window date range, vote-average sort with a vote floor). Only `/trending` (`:37-45`) is genuinely distinct (TMDB's time-decayed trending signal). After this rebuild the three redundant routes and their `lib/api.ts` fetchers (`api.ts:100-110`) have **zero remaining consumers** (verified: only `search/page.tsx` imports them) and should be deleted.
15. **`/search/person` is not proxied; `/search/movie`'s `year` param is unused** (`movies.py:70-78`).

## Part 2: Target design

One page, one mental model: **a results grid driven by the URL.** The query string is the single source of truth; every control just rewrites it.

- **Two modes, switched by the search box:**
  - **Discover mode** (empty box, the default): sort selector + collapsible Filters drawer. Default view is TMDB Trending (the strongest zero-config list); the moment any filter or non-trending sort is chosen, the page switches to `/movies/discover`. Filters compose: Genres (multi, AND), Decade (single), Min rating, Runtime bracket, Streaming service (multi, OR). The four old tabs are **replaced**: Trending and Popular and Top Rated and Newest become sort options; "Now Playing" is dropped from the core page (it's a niche theatrical list — see optional enhancement O5).
  - **Search mode** (text in the box): pure TMDB title search, same debounce. Filter controls hide behind a one-line hint ("clear the search to browse with filters") — honest, because TMDB's search endpoint accepts no genre/rating filters and client-side post-filtering is exactly the lie we're removing (finding 3). A **People** row appears under the box ("🎭 Tom Hanks · Forrest Gump"); clicking a person clears the query and pivots to discover-by-person (`with_people`), which *does* compose with every other filter.
- **Active filters are always visible** as removable gradient chips above the grid, with a real result count from `total_results`.
- **Infinite scroll** via IntersectionObserver (600px lookahead), with the Load More button kept as the sentinel's visible fallback.
- **Empty states suggest which filter to loosen** — each active filter rendered as a one-click "✕ remove" chip.

**Why replace the tabs instead of composing with them:** the four tabs were four different *rankings*, not four content sets. As a sort selector they keep their value (Trending stays the default landing experience) while collapsing the search/browse/filter matrix from `4 tabs × filters × query` to `sort × filters` or `query` — fewer states, no disappearing controls, and every state has a URL.

### Task 1 — Rewrite `backend/routers/movies.py` with `/discover` and `/person_search`

**File:** `backend/routers/movies.py` — **replace the entire file** with the following. Changes vs today: adds `GET /movies/discover` (full filter surface) and `GET /movies/person_search`; adds optional `year` to `/search`; consolidates the four duplicated httpx blocks into `_tmdb_get`; **deletes** `/popular`, `/now_playing`, `/top_rated` (finding 14 — their only consumer is the page being rebuilt); keeps `/trending`, `/{id}/providers`, `/{id}` byte-compatible.

```python
import os
from datetime import date
from typing import Optional

import httpx
from fastapi import APIRouter, HTTPException, Query

router = APIRouter()

TMDB_BASE_URL = "https://api.themoviedb.org/3"

ALLOWED_SORTS = {
    "popularity.desc",
    "vote_average.desc",
    "primary_release_date.desc",
    "revenue.desc",
}


def tmdb_key():
    key = os.getenv("TMDB_API_KEY")
    if not key:
        raise HTTPException(status_code=500, detail="TMDB_API_KEY not configured")
    return key


async def _tmdb_get(path: str, params: dict) -> dict:
    async with httpx.AsyncClient() as client:
        resp = await client.get(
            f"{TMDB_BASE_URL}{path}",
            params={"api_key": tmdb_key(), **params},
            timeout=10,
        )
    resp.raise_for_status()
    return resp.json()


@router.get("/trending")
async def get_trending(page: int = Query(1, ge=1, le=500)):
    return await _tmdb_get("/trending/movie/week", {"page": page})


@router.get("/discover")
async def discover_movies(
    page: int = Query(1, ge=1, le=500),
    sort_by: str = Query("popularity.desc"),
    genres: Optional[str] = Query(None, description="Comma-separated TMDB genre ids (AND semantics)"),
    year_gte: Optional[int] = Query(None, ge=1900, le=2100, description="Earliest release year"),
    year_lte: Optional[int] = Query(None, ge=1900, le=2100, description="Latest release year"),
    min_rating: Optional[float] = Query(None, ge=0, le=10, description="Minimum TMDB vote average"),
    runtime_gte: Optional[int] = Query(None, ge=0, le=400, description="Minimum runtime, minutes"),
    runtime_lte: Optional[int] = Query(None, ge=0, le=400, description="Maximum runtime, minutes"),
    providers: Optional[str] = Query(None, description="Comma-separated watch-provider ids (OR semantics, US region)"),
    people: Optional[str] = Query(None, description="Comma-separated TMDB person ids (cast OR crew)"),
):
    """Flexible TMDB /discover/movie proxy. Replaces the old /popular and /top_rated routes."""
    if sort_by not in ALLOWED_SORTS:
        raise HTTPException(status_code=400, detail=f"sort_by must be one of {sorted(ALLOWED_SORTS)}")

    params = {"include_adult": False, "sort_by": sort_by, "page": page}
    if genres:
        params["with_genres"] = genres
    if year_gte is not None:
        params["primary_release_date.gte"] = f"{year_gte}-01-01"
    if year_lte is not None:
        params["primary_release_date.lte"] = f"{year_lte}-12-31"
    if min_rating is not None:
        params["vote_average.gte"] = min_rating
        params["vote_count.gte"] = 200  # a 9.0 with 12 votes is noise, not a find
    if runtime_gte is not None:
        params["with_runtime.gte"] = runtime_gte
    if runtime_lte is not None:
        params["with_runtime.lte"] = runtime_lte
    if providers:
        # TMDB: pipe = OR, comma = AND. We accept commas from the client and convert to OR.
        params["with_watch_providers"] = providers.replace(",", "|")
        params["watch_region"] = "US"
        params["with_watch_monetization_types"] = "flatrate|free|ads"
    if people:
        params["with_people"] = people

    # Quality floors so sorts don't surface junk (setdefault: never override the filters above)
    if sort_by == "vote_average.desc":
        params.setdefault("vote_count.gte", 300)
    elif sort_by == "primary_release_date.desc":
        params.setdefault("vote_count.gte", 20)
        params.setdefault("primary_release_date.lte", date.today().isoformat())

    return await _tmdb_get("/discover/movie", params)


@router.get("/search")
async def search_movies(
    q: str = Query(..., min_length=1),
    page: int = Query(1, ge=1, le=500),
    year: Optional[int] = Query(None, ge=1900, le=2100),
):
    params = {"query": q, "include_adult": False, "page": page}
    if year is not None:
        params["primary_release_year"] = year
    return await _tmdb_get("/search/movie", params)


# Must be defined BEFORE /{tmdb_id} — otherwise "person_search" is captured by the
# int path param and returns 422 (same shadowing rule as /providers below).
@router.get("/person_search")
async def search_people(q: str = Query(..., min_length=2)):
    """TMDB /search/person proxy, slimmed to what the UI needs."""
    data = await _tmdb_get("/search/person", {"query": q, "include_adult": False})
    people = []
    for p in data.get("results", [])[:5]:
        known_for = [kf.get("title") or kf.get("name") or "" for kf in p.get("known_for", [])]
        people.append({
            "id": p.get("id"),
            "name": p.get("name", ""),
            "profile_path": p.get("profile_path"),
            "known_for_department": p.get("known_for_department", ""),
            "known_for": [t for t in known_for if t][:3],
        })
    return people


# Must be defined BEFORE /{tmdb_id} to avoid route shadowing
@router.get("/{tmdb_id}/providers")
async def get_movie_providers(tmdb_id: int):
    data = await _tmdb_get(f"/movie/{tmdb_id}/watch/providers", {})
    # Return US providers; fall back to empty dict
    return data.get("results", {}).get("US", {})


@router.get("/{tmdb_id}")
async def get_movie(tmdb_id: int):
    return await _tmdb_get(
        f"/movie/{tmdb_id}",
        {"append_to_response": "credits,videos"},
    )
```

Design notes for the implementer:

- **Genres are AND** (comma-joined): selecting Action + Comedy means "action comedies", the narrowing semantic a filter drawer implies. The empty state (Task 6) handles over-narrowing gracefully.
- **Providers are OR** (pipe-joined): "on Netflix or Hulu" is the only sane reading of a multi-select of services you have.
- **`sort_by` is whitelisted** — never pass arbitrary client strings through to TMDB.
- `min_rating` always pairs with a `vote_count.gte` floor; "Highest Rated" sort gets its own floor via `setdefault` so it does not override an explicit `min_rating` pairing.

```ts
export interface DiscoverParams {
  page?: number;
  sortBy?: string;
  genres?: number[];
  yearGte?: number;
  yearLte?: number;
  minRating?: number;
  runtimeGte?: number;
  runtimeLte?: number;
  providers?: number[];
  people?: number[];
}

export const discoverMovies = (p: DiscoverParams = {}) => {
  const qs = new URLSearchParams();
  qs.set("page", String(p.page ?? 1));
  if (p.sortBy) qs.set("sort_by", p.sortBy);
  if (p.genres?.length) qs.set("genres", p.genres.join(","));
  if (p.yearGte != null) qs.set("year_gte", String(p.yearGte));
  if (p.yearLte != null) qs.set("year_lte", String(p.yearLte));
  if (p.minRating != null) qs.set("min_rating", String(p.minRating));
  if (p.runtimeGte != null) qs.set("runtime_gte", String(p.runtimeGte));
  if (p.runtimeLte != null) qs.set("runtime_lte", String(p.runtimeLte));
  if (p.providers?.length) qs.set("providers", p.providers.join(","));
  if (p.people?.length) qs.set("people", p.people.join(","));
  return req<MovieListResult>(`/movies/discover?${qs.toString()}`);
};

export interface PersonResult {
  id: number;
  name: string;
  profile_path: string | null;
  known_for_department: string;
  known_for: string[];
}

export const searchPeople = (q: string) =>
  req<PersonResult[]>(`/movies/person_search?q=${encodeURIComponent(q)}`);
```

### Task 5 — Full rewrite of `app/search/page.tsx`

**File:** `frontend/app/search/page.tsx` — replace the entire file. Architecture:

- **URL = single source of truth.** Filters are *parsed* from `useSearchParams` every render; every control calls `applyFilters(next)` which `router.replace`s a new query string (`scroll: false`). The one data-fetch effect keys on `searchParams.toString()` — so typing, chip clicks, back button, refresh, and pasted links all flow through the identical path. The only local mirror is the text input (for keystroke latency) with a 400ms debounce into the URL.
- **`<Suspense>` boundary** around the inner component — required by Next.js for `useSearchParams` in production builds (see header note).
- **Staleness guard everywhere:** `fetchKeyRef` stamps every request (initial *and* load-more) with the URL it was issued for; responses for a different URL are dropped. Appends are deduped by id (finding 10).
- **Infinite scroll:** IntersectionObserver on a sentinel with `rootMargin: "600px"`; the visible Load More button doubles as manual fallback.

```tsx
            <button key={c.label} className="chip chip-active" onClick={() => applyFilters(c.clear(filters))}>
              {c.label} ✕
            </button>
          ))}
        </div>
      )}

      {error && (
        <p style={{ color: "#f87171", fontSize: "0.875rem", marginBottom: "1rem" }}>{error}</p>
      )}

      {/* Real result count from total_results — not the loaded-page count */}
      {!loading && !error && results.length > 0 && (
        <p style={{ color: "#a1a1aa", fontSize: "0.8rem", marginBottom: "1rem" }}>
          {isSearchMode
            ? `${totalResults.toLocaleString()} results for "${filters.q}"`
            : `${totalResults.toLocaleString()} movies · ${sortLabel}`}
        </p>
      )}

      {loading && <SkeletonGrid count={8} />}

      {!loading && (
        <>
          <div style={{ display: "grid", gridTemplateColumns: "repeat(auto-fill, minmax(min(320px, 100%), 1fr))", gap: "1.25rem" }}>
            {results.map((movie, i) => (
              <MovieCard
                key={movie.id}
                index={i}
                movie={{
                  id: movie.id,
                  title: movie.title,
                  poster_path: movie.poster_path,
                  vote_average: movie.vote_average ?? 0,
                  metaLine: [
                    movie.release_date ? movie.release_date.slice(0, 4) : null,
                    genreIdsToNames(movie.genre_ids).slice(0, 2).join(", ") || null,
                  ].filter(Boolean).join(" · "),
                  body: movie.overview,
                }}
                rating={ratings[movie.id] ?? 0}
                inWatchlist={watchlisted[movie.id] ?? false}
                isWatched={watched[movie.id] ?? false}
                onOpen={() => setModalId(movie.id)}
                onRate={(r) => handleRate(movie, r)}
                onWatchlist={() => handleWatchlist(movie)}
              />
            ))}
          </div>

          {/* Empty states that suggest what to loosen */}
          {results.length === 0 && !error && (
            isSearchMode ? (
              <div style={{ textAlign: "center", paddingTop: "3rem" }}>
                <p style={{ color: "#a1a1aa", marginBottom: "0.5rem" }}>No results for "{filters.q}"</p>
                {people.length > 0 && (
                  <p style={{ color: "#71717a", fontSize: "0.8rem" }}>Looking for a person? Try the People chips above.</p>
                )}
              </div>
            ) : activeCount > 0 ? (
              <div style={{ textAlign: "center", paddingTop: "3rem" }}>
                <p style={{ fontSize: "2rem", marginBottom: "0.5rem" }}>🫥</p>
                <p style={{ color: "#a1a1aa", marginBottom: "1rem" }}>
                  No movies match all of these filters. Try removing one:
                </p>
                <div style={{ display: "flex", gap: "0.4rem", justifyContent: "center", flexWrap: "wrap" }}>
                  {chips.map((c) => (
                    <button key={c.label} className="chip" onClick={() => applyFilters(c.clear(filters))}>
                      ✕ {c.label}
                    </button>
                  ))}
                </div>
              </div>
            ) : (
              <p style={{ color: "#a1a1aa", textAlign: "center", paddingTop: "3rem" }}>
                Nothing to show right now.
              </p>
            )
          )}

          {/* Infinite-scroll sentinel; the button doubles as a manual fallback */}
          {results.length > 0 && page < totalPages && (
            <div ref={sentinelRef} style={{ textAlign: "center", marginTop: "2rem", minHeight: 56 }}>
              {loadingMore ? (
                <span className="spin-icon" style={{ fontSize: "1.5rem", color: "#a855f7" }}>↻</span>
              ) : (
                <button className="btn-secondary" onClick={loadMore} style={{ minWidth: 140 }}>
                  Load More
                </button>
              )}
            </div>
          )}
        </>
      )}
    </div>
  );
}
```
