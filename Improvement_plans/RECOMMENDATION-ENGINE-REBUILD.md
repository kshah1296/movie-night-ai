## Part 1: Diagnosis (why it's broken today)

Audited files: `backend/routers/recommendations.py`, `frontend/app/page.tsx`, `frontend/lib/api.ts`, `backend/routers/movies.py`, `backend/models.py`.

### A. The "Refresh Picks" button does nothing — root cause

1. **The fingerprint cache returns identical results.** `recommendations.py:32-34` builds an MD5 fingerprint from `sorted([(r.tmdb_id, r.rating)])`. `GET /recommendations` takes **no parameters**. Clicking refresh re-calls the same endpoint with the same ratings → same fingerprint → `_cache[fingerprint]` returns the **exact same 10 movies** for 5 minutes (`CACHE_TTL = 300`). The button spins, dims the grid, and swaps in… the same list. Users read this as "broken."
2. **No cache-busting / no novelty mechanism.** There is no `seed`, no `refresh` counter, no "exclude what I just showed" param. Even if the cache were bypassed, the prompt is byte-for-byte identical, so the model returns a near-identical list.
3. **Refresh resets the genre filter** (`page.tsx:29` `setSelectedGenre(null)`), a minor extra surprise.

### B. Why the recommendations are low quality

4. **No taste weighting.** `recommendations.py:97-100` dumps every rating into the prompt with equal weight. A 1★ and a 5★ are both just "a rated movie." The prompt (`:125`) says "movies they would love" but never instructs the model to treat low ratings as **negative** signal or avoid similar films.
5. **Watchlist intent is wasted.** Watched items are used only to *exclude* (`:102-104`). "Want to watch" items (unwatched watchlist) — a strong positive signal — are ignored entirely.
6. **Fragile, hallucination-prone enrichment.** The LLM free-generates `title + year`; `_enrich_with_tmdb` (`:37-50`) does a title search and blindly takes `results[0]`. Consequences:
   - Wrong match (remakes, same-name films), or no match at all.
   - **A rec that fails to resolve has no `tmdb_id`** → its card can't be rated, can't be added to watchlist (`page.tsx:250` gates all actions on `rec.tmdb_id`). Dead card.
   - No dedupe: an enriched rec can resolve to a movie the user already rated/watched.
   - The model can invent a film that doesn't exist; only caught when enrichment silently fails (`except: pass`).
7. **Thin card data.** Recs carry only `title/year/explanation/genres/poster`. No `vote_average`, no overview, no "because you liked X" grouping — explanations are often generic.
8. **Blocking event loop.** `client.chat.completions.create(...)` (`:143`) is the **synchronous** Groq SDK called inside an `async def` route — it blocks the whole server for the duration of the LLM call. No timeout either.
9. **Genre filter is client-side over 10 items** (`page.tsx:131-132`). Selecting a niche genre frequently shows 0–2 cards.

### C. Target architecture (what we're building)

A **grounded two-stage pipeline** that makes hallucination impossible and refresh trivially correct:

```
                ┌─────────────────────────────────────────────┐
   DB ratings   │  Stage 1: Candidate generation (TMDB)        │
   + watchlist  │  • seed = loved (4–5★) + want-to-watch       │
      │         │  • TMDB /movie/{id}/recommendations + /similar│
      ▼         │  • TMDB /discover by top genres (paged)      │
  taste profile │  • exclude rated/watched/watchlisted/shown   │
      │         │  • score: seed-overlap + genre-affinity +    │
      │         │           vote_avg − disliked-genre penalty  │
      │         │  → ~30 real candidates (id, poster, votes)   │
      │         └──────────────────┬──────────────────────────┘
      ▼                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Stage 2: LLM ranks & explains (Groq, async, grounded)       │
│  • input: taste profile + the 30 candidate {id,title,...}    │
│  • output: ordered list of candidate IDs + a personal        │
│    "because you loved X" line each (NO free invention)       │
│  • map IDs back to Stage-1 objects → guaranteed poster/id    │
│  • fallback: Stage-1 score order + template explanations     │
└─────────────────────────────────────────────────────────────┘
```

Because the LLM only **selects from and explains** a real candidate set (it returns IDs we gave it), every card has a valid `tmdb_id`, poster, and vote average. **Refresh** works by passing a `seed`/`refresh` counter that (a) bypasses the cache and (b) expands the discover pages + shuffles the pool + excludes previously shown IDs → a genuinely different set every click.

### Task 1 — New taste profile + candidate generation helpers

**File:** `backend/routers/recommendations.py` (rewrite the top-of-file helpers; keep `GENRE_MAP`).

```python
import asyncio
import hashlib
import json
import os
import random
import time
from collections import Counter

from groq import Groq
import httpx
from fastapi import APIRouter, Depends, Query
from sqlalchemy.orm import Session

from backend.database import get_db
from backend.models import Rating, WatchlistItem

router = APIRouter()

TMDB_BASE_URL = "https://api.themoviedb.org/3"

GENRE_MAP = {
    28: "Action", 12: "Adventure", 16: "Animation", 35: "Comedy", 80: "Crime",
    99: "Documentary", 18: "Drama", 10751: "Family", 14: "Fantasy", 36: "History",
    27: "Horror", 10402: "Music", 9648: "Mystery", 10749: "Romance",
    878: "Science Fiction", 10770: "TV Movie", 53: "Thriller", 10752: "War", 37: "Western",
}
GENRE_NAME_TO_ID = {v: k for k, v in GENRE_MAP.items()}

# {fingerprint: (timestamp, result)} — fingerprint now includes the refresh seed
_cache: dict[str, tuple[float, list]] = {}
CACHE_TTL = 300  # 5 minutes


def _build_taste(ratings: list, watchlist: list) -> dict:
    """Split signal into loved / liked / disliked + want-to-watch, and compute genre affinity."""
    loved = [r for r in ratings if r.rating >= 4]
    liked = [r for r in ratings if r.rating == 3]
    disliked = [r for r in ratings if r.rating <= 2]
    want = [w for w in watchlist if not w.watched]

    def genres_of(items):
        ids = []
        for it in items:
            try:
                names = json.loads(it.genres) if it.genres else []
            except Exception:
                names = []
            ids += [GENRE_NAME_TO_ID[n] for n in names if n in GENRE_NAME_TO_ID]
        return ids

    loved_genres = Counter(genres_of(loved) + genres_of(want))
    disliked_genres = Counter(genres_of(disliked))
    return {
        "loved": loved, "liked": liked, "disliked": disliked, "want": want,
        "loved_genres": loved_genres, "disliked_genres": disliked_genres,
    }


async def _tmdb_get(client: httpx.AsyncClient, path: str, params: dict) -> dict:
    try:
        resp = await client.get(f"{TMDB_BASE_URL}{path}", params=params, timeout=6)
        resp.raise_for_status()
        return resp.json()
    except Exception:
        return {}


async def _candidate_pool(taste: dict, tmdb_key: str, exclude: set, seed: int) -> list:
    """Build a pool of REAL tmdb movies grounded in the user's taste. No LLM here."""
    rng = random.Random(seed)
    # Seed films: highest-rated loved movies + a couple want-to-watch, shuffled by seed
    seeds = sorted(taste["loved"], key=lambda r: r.rating, reverse=True)[:6]
    seed_ids = [r.tmdb_id for r in seeds] + [w.tmdb_id for w in taste["want"][:3]]
    rng.shuffle(seed_ids)
    seed_ids = seed_ids[:6]

    top_genres = [gid for gid, _ in taste["loved_genres"].most_common(3)]
    # discover page advances with the refresh seed so refresh pulls fresh films
    discover_page = (seed % 5) + 1

    async with httpx.AsyncClient() as client:
        tasks = []
        for sid in seed_ids:
            tasks.append(_tmdb_get(client, f"/movie/{sid}/recommendations", {"api_key": tmdb_key}))
            tasks.append(_tmdb_get(client, f"/movie/{sid}/similar", {"api_key": tmdb_key}))
        if top_genres:
            tasks.append(_tmdb_get(client, "/discover/movie", {
                "api_key": tmdb_key, "include_adult": False,
                "with_genres": ",".join(map(str, top_genres)),
                "sort_by": "popularity.desc", "vote_count.gte": 200,
                "page": discover_page,
            }))
        results = await asyncio.gather(*tasks)

    disliked_genres = set(taste["disliked_genres"])
    pool: dict[int, dict] = {}
    for data in results:
        for m in data.get("results", []):
            mid = m.get("id")
            if not mid or mid in exclude or mid in pool or not m.get("poster_path"):
                continue
            gids = m.get("genre_ids", [])
            genre_affinity = sum(taste["loved_genres"].get(g, 0) for g in gids)
            dislike_penalty = 3 * sum(1 for g in gids if g in disliked_genres)
            score = (
                genre_affinity * 2
                + m.get("vote_average", 0)
                - dislike_penalty
                + rng.uniform(0, 1.5)  # jitter so refresh reorders
            )
            pool[mid] = {
                "tmdb_id": mid,
                "title": m.get("title", ""),
                "year": int(m["release_date"][:4]) if m.get("release_date") else None,
                "genres": [GENRE_MAP.get(g, "") for g in gids if g in GENRE_MAP],
                "poster_path": m.get("poster_path"),
                "vote_average": round(m.get("vote_average", 0), 1),
                "overview": m.get("overview", ""),
                "_score": score,
            }
    return sorted(pool.values(), key=lambda x: x["_score"], reverse=True)[:30]
```

### Task 2 — LLM ranking stage (async, grounded, with template fallback)

**File:** `backend/routers/recommendations.py` (add below the Task 1 helpers).

The LLM now *picks from and explains* the candidate pool — it returns IDs we supplied, so it cannot hallucinate. The Groq call is wrapped in `asyncio.to_thread` so it stops blocking the event loop.

```python
def _taste_summary(taste: dict) -> str:
    def fmt(items):
        return [f'{r.title} ({r.year}) {int(r.rating)}★' for r in items[:12]]
    parts = []
    if taste["loved"]:
        parts.append("LOVED (4-5★): " + ", ".join(fmt(taste["loved"])))
    if taste["liked"]:
        parts.append("LIKED (3★): " + ", ".join(fmt(taste["liked"])))
    if taste["disliked"]:
        parts.append("DISLIKED (1-2★, avoid similar): " + ", ".join(fmt(taste["disliked"])))
    if taste["want"]:
        parts.append("WANTS TO WATCH: " + ", ".join(w.title for w in taste["want"][:12]))
    return "\n".join(parts)


def _groq_rank(taste: dict, candidates: list, groq_key: str, seed: int) -> dict | None:
    """Returns {tmdb_id: explanation} for the chosen subset, or None on failure."""
    client = Groq(api_key=groq_key)
    cand_lines = [
        f'{c["tmdb_id"]}: {c["title"]} ({c["year"]}) — {", ".join(c["genres"][:3])} — {c["vote_average"]}★'
        for c in candidates
    ]
    prompt = f"""You are a film curator. Here is a user's taste profile:

{_taste_summary(taste)}

Below is a numbered list of CANDIDATE movies (real TMDB ids). Pick the 12 best matches for
this user, ranked best-first. For each, write a warm, specific 1-2 sentence reason that names an
actual movie they LOVED and the concrete trait it shares (tone, director, theme, era). Avoid
anything close to their DISLIKED films.

CANDIDATES:
{chr(10).join(cand_lines)}

Respond ONLY with a valid JSON array (no markdown, no code fences). Use ONLY ids from the list:
[{{"tmdb_id": 123, "explanation": "Because you loved ..."}}]"""

    chat = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.6,
        seed=seed,
        response_format={"type": "json_object"} if False else None,  # array output; keep simple
    )
    text = chat.choices[0].message.content.strip()
    text = text.removeprefix("```json").removeprefix("```").removesuffix("```").strip()
    start, end = text.find("["), text.rfind("]")
    if start == -1 or end == -1:
        return None
    picks = json.loads(text[start:end + 1])
    valid_ids = {c["tmdb_id"] for c in candidates}
    return {p["tmdb_id"]: p.get("explanation", "") for p in picks
            if isinstance(p, dict) and p.get("tmdb_id") in valid_ids}


def _template_reason(cand: dict, taste: dict) -> str:
    """Fallback explanation when the LLM is unavailable."""
    top = sorted(taste["loved"], key=lambda r: r.rating, reverse=True)
    overlap = next((r for r in top
                    if set(json.loads(r.genres) if r.genres else []) & set(cand["genres"])), None)
    if overlap:
        return f'Shares the {", ".join(cand["genres"][:2])} vibe of "{overlap.title}", which you rated {int(overlap.rating)}★.'
    return f'A highly-rated {", ".join(cand["genres"][:2]) or "pick"} ({cand["vote_average"]}★) we think matches your taste.'
```

### Task 3 — New `GET /recommendations` route with a working `refresh` param

**File:** `backend/routers/recommendations.py` (replace the old `@router.get("/")` handler).

```python
@router.get("/")
async def get_recommendations(
    refresh: int = Query(0, description="Bump to force a fresh, different set"),
    genre: str | None = Query(None, description="Optional genre name to bias toward"),
    db: Session = Depends(get_db),
):
    ratings = db.query(Rating).all()
    if not ratings:
        return {"recommendations": [], "message": "Rate some movies first to get recommendations!", "source": "none"}

    tmdb_key = os.getenv("TMDB_API_KEY")
    groq_key = os.getenv("GROQ_API_KEY")
    if not tmdb_key:
        return {"recommendations": [], "message": "TMDB not configured.", "source": "error"}

    watchlist = db.query(WatchlistItem).all()
    taste = _build_taste(ratings, watchlist)

    # Exclude everything the user has already engaged with
    exclude = {r.tmdb_id for r in ratings} | {w.tmdb_id for w in watchlist}

    # If a genre is requested, push it to the front of the affinity counter
    if genre and genre in GENRE_NAME_TO_ID:
        taste["loved_genres"][GENRE_NAME_TO_ID[genre]] += 100

    # Cache key includes refresh seed + genre so each refresh is a distinct entry
    fp_src = json.dumps(sorted((r.tmdb_id, r.rating) for r in ratings)) + f"|{refresh}|{genre}"
    fingerprint = hashlib.md5(fp_src.encode()).hexdigest()
    now = time.time()
    if refresh == 0 and fingerprint in _cache:  # only the first (refresh=0) view is cacheable
        ts, cached = _cache[fingerprint]
        if now - ts < CACHE_TTL:
            return {"recommendations": cached, "source": "cached"}

    seed = refresh * 7919 + 17  # deterministic per refresh, varied pool/order
    candidates = await _candidate_pool(taste, tmdb_key, exclude, seed)
    if not candidates:
        return {"recommendations": [], "message": "Rate a few more movies to unlock picks.", "source": "error"}

    source = "tmdb"
    ranked = candidates  # default order = Stage-1 score
    explanations: dict[int, str] = {}

    if groq_key:
        try:
            picks = await asyncio.to_thread(_groq_rank, taste, candidates, groq_key, seed)
            if picks:
                by_id = {c["tmdb_id"]: c for c in candidates}
                ranked = [by_id[i] for i in picks if i in by_id]
                explanations = picks
                source = "ai"
        except Exception as e:
            print(f"Groq ranking failed, using TMDB order: {e}")

    out = []
    for c in ranked[:12]:
        c = {k: v for k, v in c.items() if k != "_score"}
        c["explanation"] = explanations.get(c["tmdb_id"]) or _template_reason(c, taste)
        out.append(c)

    if refresh == 0:
        _cache[fingerprint] = (now, out)
    return {"recommendations": out, "source": source}
```

**Why refresh now works:** each click sends `?refresh=N` (N increments). `refresh != 0` skips the cache entirely, and `seed` advances the discover page, reshuffles seed films, and re-jitters scoring — a genuinely different, still-personalized set every time.
