# Movie Night AI — UI/UX Audit & Implementation Plan

> Generated by a UI-expert review agent. Motto: **"A good UI is what keeps the user engaged."**
> This plan is written to be executed task-by-task by any model. Each task is self-contained with exact file paths and code. Do tasks in order — Tasks 1–4 deliver ~80% of the engagement lift.
>
> **Constraints:** keep the dark zinc theme, the `#a855f7 → #ec4899 → #f97316` gradient language, inline styles + `globals.css` utilities, CSS-only animations, **no new dependencies**. All paths relative to `frontend/`.

---

## Part 1: UI Audit

**Verdict:** The app is functional and clean, but it currently behaves like a CRUD admin panel wearing a gradient. Almost nothing moves, nothing celebrates the user's actions, and the movie posters — the single most seductive asset a movie app has — are rendered as 80px thumbnails.

### A. Engagement killers

1. **No optimistic UI anywhere — every click feels laggy.** In `app/page.tsx` (`handleRate`, `handleAddWatchlist`), `app/search/page.tsx` (`handleRate`, `handleWatchlist`), `app/watchlist/page.tsx` (`toggleWatched`, `handlePostRating`), `app/ratings/page.tsx`, and `components/MovieModal.tsx`, the code `await`s the network call **before** updating state. Stars don't fill, buttons don't flip until the round-trip completes.
2. **Zero entrance animations.** Cards in every grid pop into existence in one frame. The modal appears instantly with no fade or scale.
3. **Posters are tiny.** Every interactive page renders posters at 80×120, and `ratings/page.tsx` shrinks them to **64×96**. Only the read-only `share/page.tsx` gives posters proper 2:3 tiles — the page nobody interacts with got the best layout.
4. **Loading states are dead text.** `watchlist/page.tsx` and `ratings/page.tsx` show literal "Loading…" strings. `search/page.tsx` is worse: switching browse tabs does `setResults([])` with **no loading indicator at all** — the grid blanks out. No skeletons anywhere.
5. **"Refresh Picks" nukes the page.** Clicking refresh sets `loading=true`, which unmounts the entire grid and replaces it with the centered spinner. *(Update: Task 7 fixed the page-nuke; the deeper bug — refresh returning the identical list from the backend cache — is fixed by the backend rebuild. See `RECOMMENDATION-ENGINE-REBUILD.md` + Part 3 below.)*
6. **Rating has no reward.** `StarRating.tsx` fills stars with a flat color swap. No pop, no bounce, no moment of delight after the single most important action in the app.
7. **Destructive actions are instant and irreversible.** "Remove" on watchlist/ratings pages, plus click-to-clear stars, delete immediately with no undo.
8. **No action-failure feedback.** None of the handlers have `try/catch`. If the backend is down, the click does nothing visible.
9. **Toast vanishes without an exit.** Animates in but disappears by unmounting — a hard blink-out. A second toast fired while one is visible doesn't re-animate.
10. **The page has zero atmosphere.** Body is flat `#09090b`. For a "bold, colorful" brand, the color only lives in the logo text and chip fills.

### B. Visual design flaws

11. **Micro-font-size soup.** `0.65 / 0.7 / 0.72 / 0.75 / 0.8 / 0.85 / 0.875rem` all in active use with no scale.
12. **Inconsistent grids between sibling pages.** Home: `minmax(300px) / gap 1.25rem`. Search: `minmax(300px) / gap 1rem`. Watchlist: `minmax(340px) / gap 1rem`. Ratings: smaller 64×96 posters.
13. **Ragged card heights on home.** AI `explanation` is unclamped while search overviews clamp to 2 lines.
14. **No text hierarchy inside cards.** Metadata and the AI explanation are both `#a1a1aa` at `0.75rem` — the personalized hook of the product styled like throwaway metadata.
15. **Duplicate CSS.** `.gradient-border` declared twice in `globals.css`.
16. **`btn-primary` repeated per-card.** Every unwatched watchlist card carries a full gradient "✓ Mark Watched" button — 20 cards = 20 loud gradients.
17. **Watched cards dim everything** (`opacity: 0.75` on whole card), washing out the title and the user's own rating.
18. **`@keyframes spin` lives inline in `page.tsx`** via a `<style>` tag instead of `globals.css`.

### C. Interaction flaws

19. **Chips, tabs, and nav links have no hover states.** They're inline-styled buttons — inline styles can't express `:hover`, so they sit inert under the cursor.
20. **Dead click: clearing a post-watch rating does nothing.** `watchlist/page.tsx` — `if (rating === 0) return;` while StarRating's aria-label promises "click to clear."
21. **Invalid HTML in the modal trailer:** `<button>` nested inside `<a>`.
22. **Search input's `autoFocus`** yanks the mobile keyboard open on page load.
23. **Layout shift from the results-count line** — always renders even when empty.
24. **Hover affordance with no action on `/share`** — read-only tiles use `gradient-border` which lifts on hover, promising interactivity.
25. **Star tap targets are too small.** `sm` is ~18px with no padding — far below the ~44px touch minimum.

### D. Mobile / responsive

26. **The nav breaks on small screens.** Brand + four links in a no-wrap 60px flex row overflow at 375px viewport.
27. **Toast overflows on mobile.** `whiteSpace: "nowrap"` — long messages run wider than the screen.
28. **Genre chip rows can stack 4–5 deep on phones**, pushing content below the fold.

## Part 2: Implementation Plan

### Task 1 — Optimistic UI + error handling for all rate/watchlist actions

**Files:** `app/page.tsx`, `app/search/page.tsx`, `app/watchlist/page.tsx`, `app/ratings/page.tsx`, `components/MovieModal.tsx`

For **every** handler that currently does `await apiCall(); setState(...)`, invert the order: snapshot previous state → update state immediately → `await` inside `try` → on `catch`, restore the snapshot and show an error toast.

Exact pattern (example for `handleRate` in `app/page.tsx`; replicate in each handler listed below):

```tsx
async function handleRate(rec: Recommendation, rating: number) {
  if (!rec.tmdb_id) return;
  const id = rec.tmdb_id;
  const year = typeof rec.year === "number" ? rec.year : null;
  const prevRating = ratings[id] ?? 0;
  const prevWatchlisted = watchlisted[id] ?? false;

  // optimistic update FIRST
  if (rating === 0) {
    setRatings((prev) => { const n = { ...prev }; delete n[id]; return n; });
    setToast(`Removed rating for "${rec.title}"`);
  } else {
    setRatings((prev) => ({ ...prev, [id]: rating }));
    setWatchlisted((prev) => ({ ...prev, [id]: true }));
    setToast(`Rated "${rec.title}" ${rating}★ · Added to Watched`);
  }

  try {
    if (rating === 0) await deleteRating(id);
    else await rateAndAddWatched({ tmdb_id: id, title: rec.title, poster_path: rec.poster_path, genres: rec.genres, year, rating });
  } catch {
    // revert
    if (prevRating === 0) setRatings((prev) => { const n = { ...prev }; delete n[id]; return n; });
    else setRatings((prev) => ({ ...prev, [id]: prevRating }));
    setWatchlisted((prev) => ({ ...prev, [id]: prevWatchlisted }));
    setToast("Couldn't save — is the backend running?");
  }
}
```

Apply to: `handleRate` + `handleAddWatchlist` (`page.tsx`), `handleRate` + `handleWatchlist` (`search/page.tsx`), `toggleWatched` + `handlePostRating` + `handleRemove` (`watchlist/page.tsx` — for `toggleWatched`/`handlePostRating`, optimistically patch with `setItems(prev => prev.map(i => i.tmdb_id === item.tmdb_id ? { ...i, watched: !item.watched } : i))` then reconcile with the server's `updated` object on success), `handleRatingChange` (`ratings/page.tsx`), and `handleRate` + `handleWatchlist` (`MovieModal.tsx`). Do not change function signatures or props.

**8b.** Rewrite `Toast.tsx`:

```tsx
"use client";

import { useEffect, useState } from "react";

interface Props {
  message: string;
  onDismiss: () => void;
  duration?: number;
  actionLabel?: string;
  onAction?: () => void;
}

export default function Toast({ message, onDismiss, duration = 2500, actionLabel, onAction }: Props) {
  const [leaving, setLeaving] = useState(false);

  useEffect(() => {
    if (!message) return;
    setLeaving(false);
    const t1 = setTimeout(() => setLeaving(true), duration - 200);
    const t2 = setTimeout(onDismiss, duration);
    return () => { clearTimeout(t1); clearTimeout(t2); };
  }, [message]); // eslint-disable-line react-hooks/exhaustive-deps

  if (!message) return null;

  return (
    <div
      key={message}
      style={{
        position: "fixed", bottom: "2rem", left: "50%", transform: "translateX(-50%)",
        background: "linear-gradient(135deg, #a855f7, #ec4899)", color: "white",
        padding: "0.6rem 1.5rem", borderRadius: "999px", fontWeight: 600,
        zIndex: 9999, fontSize: "0.875rem", textAlign: "center",
        maxWidth: "calc(100vw - 2rem)",
        display: "flex", alignItems: "center", gap: "0.75rem",
        animation: leaving ? "toast-slide-down 0.2s ease-in both" : "toast-slide-up 0.2s ease-out",
        pointerEvents: actionLabel ? "auto" : "none",
      }}
    >
      <span>{message}</span>
      {actionLabel && onAction && (
        <button
          onClick={() => { onAction(); onDismiss(); }}
          style={{ background: "rgba(255,255,255,0.2)", border: "none", color: "white",
                   borderRadius: "999px", padding: "0.2rem 0.7rem", fontSize: "0.75rem",
                   fontWeight: 700, cursor: "pointer" }}
        >
          {actionLabel}
        </button>
      )}
    </div>
  );
}
```

Note: `whiteSpace: nowrap` removed (fixes mobile overflow); `key={message}` replays the entrance for consecutive toasts.
